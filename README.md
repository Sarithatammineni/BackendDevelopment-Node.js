Q1. Role of frontend
Frontend is responsible for users see, interact with, and experience in their browser or device.

⦁	User Interface: Creates the UI users interact with layout ,design.
⦁	User Interaction: It handles clicks, inputs, navigation.
⦁	Communication with backend : Makes API calls to fetch data.




Q2. Role of backend
Backend process requests, apply business rules, manage data, and ensure security.

⦁	Server -side processing: Processes request, applies business logic.
⦁	Database Handling: Stores, retrieves , updates data in databases.
⦁	Security and Authentication: Manages logins, permissions, protects data.




Q3.Business Logic

Business logic specific to the business app

Ex: E-commerce-Applying discounts based on valid coupon codes
Banking- Validating transactions before withdrawal
Social media- Showing posts based on user preferences



Q4. Client-Server Model

The Client–Server Model is a network architecture in which clients request services or resources, and servers provide them.

⦁	Client- The user's browser makes requests
⦁	Server- Backend that processes requests and sends responses
⦁	Communication- Client sends HTTP request -> server processes -> Sends response -> Client renders




Q5. Three-Tier Architecture-> It is a software design pattern
3-tier architecture is used because it makes web applications secure and efficient.

⦁	Presentation layer-It interacts with the user (frontend)
⦁	Application Tier- It processes  user requests(backend)
⦁	Data Tier- It stores and manages data(database layer)



Q6. JavaScript as backend language
JavaScript is used as backend language because it enables fast, scalable and efficient server -side development .

⦁	Performance: Efficient for handling thousands of connections
⦁	Ecosystem: npm is the largest package manager
⦁	Popular backend frameworks: Express.js , NestJS
